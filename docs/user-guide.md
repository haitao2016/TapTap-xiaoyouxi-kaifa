# TapDev Studio 用户指南

## 简介

TapDev Studio 是一款专为 TapTap 小游戏开发者打造的集成开发环境（IDE），提供代码编辑、调试、监控、构建等一站式开发体验。

## 快速开始

### 安装与运行

```bash
# 安装依赖
pnpm install

# 启动开发服务器（Web模式）
pnpm dev

# 启动桌面端（完整功能）
pnpm dev:desktop
```

### 打开项目

1. 点击左侧导航栏的「项目」菜单
2. 选择「打开项目」或创建新项目
3. 选择 Unity 项目目录

## 功能模块

### 代码编辑器

TapDev Studio 集成了 Monaco 代码编辑器，支持：
- 语法高亮
- 代码补全
- 多标签页编辑
- 资源管理器

### 调试功能

#### 启动调试服务器

1. 点击左侧导航栏的「调试」菜单
2. 点击「启动调试服务器」
3. 使用手机扫描二维码连接

#### 调试功能
- 实时日志查看
- 断点管理
- 变量监视
- 性能分析

### 性能监控

点击左侧导航栏的「监控」菜单，实时查看：
- **FPS**: 帧率监控
- **内存**: 内存使用量
- **CPU**: CPU 使用率
- **GPU**: GPU 内存
- **网络**: 网络请求统计

#### 告警系统
当性能指标超过阈值时会触发告警：
- FPS < 30：警告
- 内存使用率 > 85%：严重
- CPU > 90%：警告

### 构建发布

点击左侧导航栏的「构建」菜单：

#### 选择平台
- **WebGL**: 网页端构建
- **Android**: Android 原生构建（需要桌面端）
- **iOS**: iOS 原生构建（需要桌面端）

#### 构建选项
- **压缩打包**: 生成 game.zip
- **WASM 分包**: 生成 game_wasm_split.zip
- **资源优化**: 优化 Unity 资源
- **移除调试信息**: 减小包体积

#### 构建流程
1. 配置构建选项
2. 点击「Unity 构建」或「模拟构建」
3. 等待构建完成
4. 查看输出文件

### 插件系统

点击左侧导航栏的「插件」菜单：

#### 已内置插件
- **Unity 工具集**: 资源优化、缓存清理
- **网络测试**: CDN 测试、Ping 测试
- **调试工具**: 性能快照、调用栈查看

#### 命令面板
使用快捷键 `Ctrl+P` 打开命令面板，快速搜索和执行命令。

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+P` | 打开命令面板 |
| `Ctrl+S` | 保存文件 |
| `Ctrl+Shift+P` | 性能快照 |
| `Ctrl+Shift+O` | 优化资源 |
| `Ctrl+Shift+Delete` | 清除缓存 |

## 平台支持

### Web 模式
- 支持所有现代浏览器
- 功能：编辑器、监控、模拟构建
- 限制：无法进行真实 Unity 构建

### 桌面端（Electron）
- 支持 Windows、macOS、Linux
- 功能：完整功能 + Unity 真实构建
- 需要安装 Unity 编辑器

### 移动端（Capacitor）
- 支持 Android、iOS
- 功能：调试、监控
- 限制：不支持构建

## 项目配置

项目配置文件位于项目根目录的 `.tapdev/config.json`：

```json
{
  "projectId": "my-game",
  "name": "My Game",
  "version": "1.0.0",
  "buildPath": "./Builds",
  "cdnUrl": "https://cdn.example.com",
  "unityPath": "auto",
  "monitor": {
    "fpsThreshold": 30,
    "memoryThreshold": 0.85
  }
}
```

## 常见问题

### Q: Unity 构建失败怎么办？

A: 请检查：
1. Unity 是否已正确安装
2. TapTap SDK 是否已导入项目
3. 构建路径是否正确
4. 是否有足够的磁盘空间

### Q: 调试连接失败？

A: 请检查：
1. 手机和电脑是否在同一网络
2. 防火墙是否允许连接
3. 调试服务器是否已启动

### Q: 监控数据不更新？

A: 请确保：
1. 监控已启动（点击「开始监控」）
2. 网络连接正常
3. 浏览器控制台没有错误
